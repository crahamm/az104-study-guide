1. If a blob in the Cool tier has not been modified or accessed for 90 days according to a lifecycle management policy, what should be the next action?

2. Your organization wants to ensure data is accessible in the Hot tier for two months, then moved to the Cool tier if not accessed, and finally archived after one year. What is a key benefit of this lifecycle policy?
  
  - It balances data accessibility with cost savings over time.

3. An organization wants to use Azure Blob Storage to ensure compute workloads have access to consistent data across regions. Which configuration should they implement to achieve this?

  - Set up object replication to ensure the same blobs are accessible in multiple regions.

4.Which lifecycle management policy rule can be used to reduce storage costs for data that is rarely accessed after a month?

  - Move data to Archive tier

5. What should happen according to a lifecycle management policy if a blob has not been accessed for over 365 days?

  - The blob should be moved to the Archive tier for long-term storage cost savings.

6. When organizing blobs in Azure Blob Storage, what is the significance of container naming conventions?

  - Ensures uniqueness within the storage account and aids in the organization of data.

7. If object replication is configured but snapshots are not being replicated, what is the most likely cause of this issue in Azure Blob Storage?

  - Object replication does not support snapshot replication.

8. How can lifecycle management policy rules automate the transition of data to cooler storage tiers in Azure Blob Storage?

  - By defining rule-based conditions that transition data based on its age and access frequency.

9. What might prevent successful object replication between two Azure Blob Storage accounts?

  - Blob versioning is not enabled on either the source or destination account.
