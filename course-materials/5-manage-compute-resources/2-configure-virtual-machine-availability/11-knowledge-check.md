1. Your company needs to ensure high availability for its critical application with minimal data center failures. Which Azure feature should you implement?

- Availability Zones

2. Which Azure feature should be used to ensure that a group of virtual machines remains available even if a specific data center within a region fails?

- Availability Zones

3. Which of the following is a limitation of vertical scaling compared to horizontal scaling when managing virtual machine resources?

- Limited by maximum hardware capacity

4. If a virtual machine hosting a memory-intensive application needs to accommodate growing data loads, which scaling method should be prioritized to improve performance?

- Vertical scaling by increasing RAM

5. Your company wants to minimize costs while maintaining performance during peak times. Which scaling strategy is more suitable for dynamically adjusting resource allocation in response to varying workloads?

- Horizontal scaling with Azure Virtual Machine Scale Sets

6. Which scaling strategy should be adopted for a microservices architecture to ensure each service can scale independently based on demand?

- Horizontal scaling by deploying each microservice in separate containers or VMs

7. Your company is designing a new application that must remain operational even if an entire Azure datacenter fails. Which strategy should be employed to meet this requirement?

- Distribute application components across multiple availability zones within a region.

8. What is a key advantage of using Azure Virtual Machine Scale Sets for managing application demand fluctuations?

- Automatic adjustment of virtual machine count as demand changes.

9. Your company experiences fluctuating workloads on its web application throughout the day. Which Azure feature should you implement to dynamically adjust the number of virtual machine instances based on workload demands?

- Autoscale with Azure Virtual Machine Scale Sets
