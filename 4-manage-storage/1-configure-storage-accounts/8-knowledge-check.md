1. When comparing Azure's Locally Redundant Storage (LRS) and Geo-Redundant Storage (GRS), what is the main advantage of choosing GRS?

- Data is replicated to a secondary geographic region to protect against regional outages.

2. Which of the following statements accurately describes the function of Azure Private Endpoints in the context of Azure Storage?

- Azure Private Endpoints enable secure connections by assigning a private IP from your VNet to the Azure Storage account.

3. A customer wants to map their custom domain to an Azure Blob Storage account. Which Azure feature allows the customer to achieve this?

- Azure Custom Domain

4. An Azure administrator needs to secure the storage account by restricting access to specific subnets. What configuration should they apply in the Azure portal?

- Use the 'Firewalls and virtual networks' settings for the storage account.

5. Your company wants to ensure that blob data in Azure Storage can be accessed through a custom domain securely. Which protocol should you ensure is enabled to maintain secure access?

- HTTPS

6. What is a common mistake when setting up a custom domain for Azure Blob Storage that can lead to access issues?

- Not verifying the domain ownership before mapping it to the storage account.

7. Which replication strategy would be most appropriate for data that must remain accessible during a regional outage, but does not require immediate consistency for all operations?

- Geo-redundant storage (GRS)

8. How does Read-access geo-redundant storage (RA-GRS) improve upon standard Geo-redundant storage (GRS) in terms of data access?

- RA-GRS allows read access to data in the secondary region, enhancing availability during primary region failures.
